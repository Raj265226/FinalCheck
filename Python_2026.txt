Object-oriented Programming - It's a style of programming where template is created and copy of the template is
called Object Oriented Programming.  The template is called Class and the copies created from it is called Object

Constructor - While creating an object, the special method __init__() inside the class is invoked automatically.
this special method is called as constructor. Example -
class Mobile:
    def __init__(self,brand):
        print('Inside constructor')
        self.brand = brand
mob1=Mobile('Apple')                            --          Inside constructor
print('Mobile name - ', mob1.brand)                         Mobile name -  Apple

Encapsulaion - Adding __ makes the scope of the attribute as private. Private attributes are those which are accessible
               only inside the class. This method of restricting access to our data is called Encapsulation.
        Private variable can be accessed outside the class after modified it to _Class__attribute
    class Customer:
        def __init__(self,name,balance):
            self.name = name
            self.__balance = balance
        def update_balance(self,amount):
            if amount < 1000 and amount > 0:
                self.__balance += amount
        def show_balance(self):
            print('Balance is: ',self.__balance)
    c1 = Customer('Raj',1000)
    print(c1.name)                  Condition 1     --      Raj
    print(c1.__balance)             Condition 1     --      AttributeError
    c1.__balance = 100              Condition 2     --      It will not affect
    c1.update_balance(500)          Condition 2
    c1.show_balance()               Condition 2     --      1500
    c1._Customer__balance = 100     Condition 3     --      It will affect/update
    c1.update_balance(500)          Condition 3
    c1.show_balance()               Condition 3     --      600

Getter and Setter Methods - To have an error free way of accessing and updating private variables, specific methods
                            are created for this.
       Methods which are meant to set a value to a private variable are called Setter/Mutator methods.
       Methods meant to access private variable values are called Getter/Accessor methods.
    class Customer:
        def __init__(self,name,balance):
            self.name = name
            self.__balance = balance
        def set_balance(self,amount):
            if amount < 1000 and amount > 0:
                self.__balance = amount
        def get_balance(self):
            return self.__balance
    c1 = Customer('Raj',1000)
    c1.set_balance(100)
    print(c1.get_balance())             --      100

Accessing Class Methods - It can be accessed directly by using class name, even without creating objects.
    class Mobile:
        __discount = 50
        def __init__(self,price):
            self.price=price
        @classmethod    
        def get_discount(cls):
            return cls.__discount
        @classmethod 
        def set_discount(cls,discount):
            cls.__discount = discount
    print(Mobile.get_discount())            --      50

Type of OOP Relationships -
Inheritence - When one object is a type fo another object. Ex - Mobile is a product
Aggregation - When one object owns another object, but they both have independent life cycle. Ex - Customer has a address, 
              If customer is no more, other customer will be available on that address.
Composition - When one object owns another object, but they both have same life cycle. Ex - College has a dept. If college closes
              dept is also closed

Inheritance - When a class inherits from another class, then those classes are said to have an inheritance relationship. The 
              class which is inheriting is called the child/sub/derived class and the class which is getting inherited is 
              called the parent/super/base class.
Inherited feature in child class - A child class inherits Constructor , Non-privste Attribtes and Methods. Private variables 
              get inherited in Python.
Private attributes and Inheritance - The child class has every access that parent has. But child class can't directly access 
                private attributes of parent class. Here also need to define setter and getter.
    class Phone:
        def __init__(self,price,brand):

            self.brand = brand

Using custom exception - custom exceptions can be used and handled differently in except block
    class InvalidPrice(Exception):
        pass
    class WrongCard(Exception):
        pass
    class Creditcard:
        def __init__(self,card_no,balance):
            self.card_no = card_no
            self.balance = balance
    class Customer:
        def __init__(self,cards):
            self.cards = cards
        def purchase_item(self,price,balance):
            if price < 0:
                raise InvalidPrice('Price is wrong')
            if card_no not in self.cards:
                raise WrongCard('Invalid card')
            if price > self.cards[card_no].balance:
                raise WrongCard('Insuff balance')
    card1=Creditcard(101,800)
    card2=Creditcard(102,2000)
    cards = {card1.card_no:card1,card2.card_no:card2}
    c=Customer(cards)
    while(True):
        card_no=int(input('Enter card_no:'))        --      101 - Insuff balance / 102 / 103 - Invalid card
        try:
            c.purchase_item(1200,card_no)
            break
        except InvalidPrice as e:
            print(str(e))
            break
        except WrongCard as e:
            print(str(e))
            continue
        except Exception as e:
            print('Something went wrong' + str(e))

Custom exeptions with constructor overriding - custom exception class can override the default constructor provided by 
                    the Exception class.
    class InvalidUsername(Exception):
        def __init__(self,username):
            msg = 'The given username '+username+' is invalid'
            super().__init__(msg)
    try:
        username = '1abd'
        if not username[0].isalpha():
            raise InvalidUsername(username)
    except InvalidUsername as e:
        print(e)                                --      The given username 1abd is invalid

Order of except block - The except block of Exception is executed, even through InvalidPrice is raised, It's because 
        a parent class except block  will be able to handle all the child class except blocks also. Since Exception 
        is a parent class to InvalidPrice, it can handle the exception of type InvalidPrice
    class InvalidPrice(Exception):
        pass
    try:
        raise InvalidPrice()
    except InvalidPrice:
        print('InvalidPrice')
    except Exception as e:
        print('Exception')

Special method - __str__ is a special method and if we use it then Object will be printed. it always return string
    class Product:
        def __init__(self,name,price):

        def __str__(self):

Add method for Operator Overloading - __add__(self,other) is used to define the functionality of + operator.

    class Product:
        def __init__(self,name,price):
            self.name=name
            self.price=price
        def __add__(self,other):
            return self.price+other.price
    p1 = Product('Iphone',80000)
    p2 = Product('Android',20000)
    print('total cost -',p1+p2)                      --      total cost - 100000
